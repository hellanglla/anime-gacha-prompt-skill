# UI类型规范与防崩指南

## UI类型描述

### 角色图鉴UI (Character Profile)

```
完整游戏UI叠层，左侧角色稀有度与信息面板，巨大金色SSR发光字样、星级、属性图标；右侧基础属性、技能图标、专属武器面板；底部costume/skin切换栏与功能按钮。UI边框为黑金哥特雕花风，镶嵌赤红裂纹与暗纹，像高端二次元手游角色详情页截图。文字为中文 fantasy UI typography，带有精美装饰性字体设计。
```

### 角色立绘展示UI (Character Card)

```
完整的游戏角色卡牌界面，左上角显示稀有度标识与星级，下方显示角色名称。左侧占据大部分画面的大型角色立绘展示区，展示角色全身立绘与完整背景场景。右侧排列多个小方框展示角色不同角度特写。底部显示角色介绍文本框。UI边框为黑金哥特雕花风，镶嵌赤红裂纹与暗纹。文字为中文 fantasy UI typography，带有精美装饰性字体设计。
```

### 抽卡UI (Gacha Pull)

```
华丽的抽卡登场界面，角色从金色与紫色光芒中登场，周围环绕光效粒子和能量流。稀有度标识醒目展示。底部显示角色名称与属性标签。整体充满动态感和视觉冲击力，UI边框为黑金哥特雕花风。文字为中文 fantasy UI typography，带有精美装饰性字体设计。
```

## 高风险姿势（容易出崩坏）

以下姿势结构复杂，AI绘画容易产生身体比例失真：

| 姿势 | 风险 | 缓解措施 |
|------|------|---------|
| 趴姿 | 四肢比例易崩、手部扭曲 | 降低细节复杂度至B档，面部用最高优先级 |
| 倒挂 | 身体比例完全失真 | 不建议单独使用，建议降低细节至A档 |
| 蜷缩 | 肢体粘连、形状混乱 | 使用清晰构图，镜头不可太远 |
| 漂浮 | 身体重力感丢失 | 在描述中加上明确的身体方向和支撑关系 |
| 劈叉 | 腿部结构崩坏 | 降低动态等级，或使用闭合双腿描述 |

---

## 防崩通用策略

1. **高风险姿势** → 降低细节复杂度（选A或B，避免C）
2. **镜头越近** → 崩坏概率越低（特写和近景比全身安全）
3. **静态比动态安全** → 不确定效果时选0级或1级
4. **面部始终设为最高细节优先级**

---

## 冲突检测规则

以下组合必须警告用户：

| 条件 | 警告内容 |
|------|---------|
| 选了趴姿/倒挂 + 细节C(复杂) | "⚠ 趴姿/倒挂配合极致复杂细节容易崩坏，建议降低细节至B档或A档" |
| 选了漂浮+动态2-3 | "⚠ 漂浮配合高动态容易丢失身体结构，建议限制在动态1级" |
| 选了全身(D4)+细节C | "⚠ 全身远景配合极致细节容易崩，建议用中景(D3)或标准细节(B)" |
| 抽卡UI+静态姿势(0) | "⚠ 抽卡UI需要动态感，当前姿势为静态，建议提升至动态2级" |
